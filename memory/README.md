# 비서의 뇌 (memory/)

여기는 **포트폴리오 매니저 비서의 장기기억**이다. `CLAUDE.md`(프로토콜)·`PROJECTS.md`(스냅샷)와 함께 이 루트에서 일하는 나(Claude)의 총체적 의사결정 기록을 담는다.

**보인다. 안 휘발된다.** 이건 채팅 컨텍스트가 아니라 디스크 위의 마크다운 파일이다. 세션이 끝나도, 컨텍스트가 압축돼도, 맥을 껐다 켜도 그대로 남는다. 사라지는 유일한 경우는 네가 파일을 지울 때뿐.

## 무엇을 기억하고, 무엇을 기억하지 않는가

기억을 성질별로 갈라서 **재생성 불가한 것만** 저장한다:

| 성질 | 예시 | 여기 저장? |
|---|---|---|
| 재생성 가능 | 레포 파일 트리, 커밋 모멘텀 | ❌ 필요할 때 git으로 즉석 생성 |
| 천천히 변함 | 아키텍처 지도(핵심 디렉토리·엔트리포인트) | 🔶 `architecture/`에 온디맨드 |
| **재생성 불가** | **의사결정·이유·이탈·유저 성향** | ✅ 여기가 핵심 |

레포 구조가 궁금하면 `git ls-files`로 뽑으면 된다 — 그건 기억할 게 아니라 **왜 그렇게 만들었는지(결정)**를 기억한다.

## 구조

```
memory/
├── README.md            ← 이 파일
├── preferences.md       ← 유저 성향 · 매니저 행동 규칙
├── decisions/           ← ⭐ 의사결정 로그 (append-only, 최신이 위)
│   ├── portfolio.md        루트/크로스프로젝트 차원의 결정
│   ├── ant-index.md
│   ├── zentarot.md
│   └── bumang-blog.md
└── architecture/        ← 레포별 아키텍처 지도 (필요할 때 생김)
```

## 규칙

- **한 결정 = 한 블록.** 최신을 맨 위에 append. 과거 결정은 지우지 말고 상태만 갱신(확정→폐기 등).
- **결정엔 반드시 이유를 붙인다.** 나중의 나/너가 "왜 이렇게 갔지"를 되물을 때 답이 여기 있어야 한다.
- 결정끼리 연결은 `[[파일명]]`으로 (예: `[[zentarot]]`).
- 프로토콜 전문은 루트 `CLAUDE.md`의 **"비서 프로토콜"** 섹션 참조.
