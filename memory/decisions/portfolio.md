### 2026-07-11 · 방법론 전환 — "코드 제로투원"에서 "기획 구체화 우선"으로
- **결정**: 새 사이드 프로젝트를 곧장 코드로 제로투원 하려던 성향을 버린다. **기획을 먼저 구체화**하고, 각 프로젝트별로 **기획 히스토리를 축적**한다. 코드 착수는 기획이 여문 뒤.
- **이유**: (유저 소견) 지금까지 바로 코드부터 짜는 데 집착했지만, 기획 구체화가 더 중요하다는 판단. **관찰 신호가 이를 뒷받침**: ant-index(6주 휴면)·zentarot(3주+ 멈춤) 둘 다 코드 PoC까진 갔으나 기획·에셋·방향이 안 여물어 정체. "코드 먼저"의 대가가 포트폴리오에 찍혀 있다.
- **축적 위치 (확정)**: **각 프로젝트 repo의 `<project>/PLANNING.md`** — 최신이 위로 오는 날짜순 append(`## YYYY-MM-DD · 제목` / 문제 → 선택지 → 결정·이유). 코드와 함께 버전관리되고 "정본은 현장"(2026-07-06 결정)과 일관. 매니저는 on-demand로 참조. **하이브리드**: repo가 아직 없는 초기 아이디어는 매니저 memory에 임시로 두다가 repo 생기면 그 repo의 PLANNING.md로 이관.
- **기획 결정 vs 실행 결정**: 기획(무엇을·왜 만들지, 방향·범위)은 `PLANNING.md`. 이미 하는 방식대로의 실행·아키텍처 결정은 각 repo의 `CLAUDE.md`(커밋 해시 첨부). 매니저 크로스프로젝트 결정은 계속 이 파일.
- **정본 이관 완료(2026-07-11)**: 3개 repo에 `PLANNING.md` 스캐폴딩하고, 기존에 매니저 memory에 고여 있던 프로젝트별 핵심 결정을 각 repo의 PLANNING.md로 옮겼다. 매니저 `memory/decisions/{ant-index,zentarot,bumang-blog}.md`는 **포인터+종합**으로 축소(정본 이중화 제거). 이 파일(portfolio)만 크로스프로젝트 정본으로 남는다.
- **상태**: 확정 (방향 + 메커니즘 + 이관 완료).
- 연결: [[../preferences]]

### 2026-07-06 · 맥락/도구 경계 — "얇은 렌즈 + 정본은 현장"
- **결정**: 루트 매니저는 **얇은 고고도 렌즈**로 확정. 하위 레포 맥락을 미러링·연속 sync하지 않는다.
  1. **스킬·훅은 제자리** — 프로젝트 전용 도구(블로그의 `write-blog-post`·`deploy-bumang`·`segment-draft`)는 해당 레포 `.claude/`에. 루트로 옮기거나 링킹하지 않음. 판정 기준 = **"프로젝트 내부냐 가로지르냐"**(내부→그 레포, 가로지름→루트).
  2. **작업은 도구가 사는 곳에서** — 프로젝트 내부 작업은 하위 세션(스킬·훅이 cwd-scoped라 거기서만 발화), 커밋/푸시 오케스트레이션·상담은 루트.
  3. **decision 두 집** — 포트폴리오 결정은 루트가 직접 씀(이 파일), 프로젝트 결정은 하위 레포 `CLAUDE.md`에 한 줄씩(블로그가 이미 하는 방식, 커밋 해시 첨부).
  4. **루트는 on-demand로 읽는다** — git 로그 + 하위 `CLAUDE.md`를 상담 시 즉석 생성. 저장(미러)하지 않음.
  5. **자동화는 필요 증명 후** — `/wrap` 증류 스킬·`/portfolio-brief` 다이제스트는 보류. "하위에서 정했는데 루트가 몰랐네" 순간이 **3번+ 쌓이면** 그때 제작(그 3번이 무엇을 담을지 알려줌).
  6. **캡처 정본 하나** — bespoke `memory/`로 통일. workbook(`wb-knowledge`/`wb-session`)을 개인 레포에 겹쳐 쓰지 않음(정본 이중화 = 최대 손실 위험).
- **이유**: (a) "손실없이 raw 축적"은 틀린 목표 — 가치는 증류물(결정·이유·함정)이지 과정 스트림이 아님. (b) 매니저가 얇은 건 정상; 문제는 얇음이 아니라 *결정*이 새는지 여부. (c) Claude Code 스킬·훅은 cwd-scoped라 루트에서 하위 도구를 온전히 못 씀 → 작업축과 맥락축을 분리하면 해소. (d) 지금 상시 캡처를 지으면 무엇을 담을지 모른 채 그릇부터 만드는 셈(YAGNI). 본진은 개미지표·타로·블로그, 인프라는 본진이 아플 때만.
- **상태**: 확정 · 인프라 동결
- 연결: [[../preferences]]

### 2026-07-06 · GitHub 멀티계정 — 개인 레포 push는 계정 전환 필수

HTTPS 자격증명 우선순위 = ① `GITHUB_TOKEN` env → ② gh 키링 활성계정. 그래서 그냥 push하면 **403 (Permission denied to bhjeong-camfit)**.

- **주의**: SSH 키(`id_ed25519`)도 **회사 계정** 거라 SSH 전환은 답이 아님.
- **커밋 identity**: 이 루트 하위 레포는 **전부 `Bumang-Cyber <calmness0729@gmail.com>`**로 통일(2026-07-06 사용자 지정). 각 레포에 `user.name/email`을 레포별 config로 박아둠 → 그냥 커밋해도 자동으로 맞음. 새 레포 추가 시 동일 config 설정. (ant-index의 회사 identity 커밋 2개는 재작성+force-push로 교정 완료.)
